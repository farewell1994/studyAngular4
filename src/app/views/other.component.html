Замітки, і те на що варто звернути увагу

  • Форми для сторінки факторіал побудовані Data-Driven підходом, а для головної - Template-Driven
  • Реалізована валідація через класи input елементів і html pattern
  • Редірект при спробі зайти на сторінку /admin на сторінку /page
  • Для входу на сторінку Guard потрібно ввести "user"
  • На головній задіяний pipe для переведення строки у верхній регістр, та власностворений pipe, який знаходить факторіал числа
  • На сторінці parent задіяна власностворена директива, що змінює шрифт
  • Нижче розміщена кнопка, яка змінює background (ngClass)
